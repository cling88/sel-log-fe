# Sellog 서비스 가이드 (구현 기준)

---

## 문서 목적

- 이 문서는 **현재 FE 퍼블 구현 상태**를 기준으로 작성한다.
- 추후 BE 전달 시, "이미 구현됨 / 미구현(보류)"를 구분해 혼선을 줄이는 용도다.
- 아래에 없는 항목은 현재 기준 미구현으로 본다.

---

## 공통

**GNB:** 대시보드 / 장부 / 설정  
**장부 탭:** 매입 / 매출 / 수익 / 상품관리

### 장부 헤더 (정산 추이)

- 상단 `정산 추이` 영역은 접기/펼치기 가능
- 프리셋 버튼: `전체 / 올해 / 이번달 / 주간 / 오늘` (UI만 존재, 실제 필터 연동 미구현)
- **카드 3개 표시: `매입 / 매출(정상) / 수익(입금)`**

**정산 추이 계산 기준**

> 각 탭별 데이터의 **전체 최종값**만 합산한 값.  
> 기간 필터 없이 전체 데이터 기준.

| 카드 | 계산 공식 |
|------|-----------|
| 매입 | 상품매입 합계 + 공급비용 합계 (기타지출 제외) |
| 매출 (정상) | `status = "normal"` 주문의 `totalAmount` 합계 |
| 수익 (입금) | 입금 내역의 `amount` 합계 |

- 추이 하단 요약:
  - `수익 입금 N원`
  - `- 누적 기타지출 M원` (기타지출 탭 전체 합계)
  - `= 총 K원`
- 퍼블 기준: 시드 데이터로 계산 → BE 연동 시 API 집계로 교체

### 월 선택 표시 정책

- `매입/매출/수익` 탭: 월 선택 버튼 표시
- `상품관리` 탭: 월 선택 버튼 숨김
- 현재 월 선택은 표시용이며, 실제 데이터 필터 연동은 미구현

### 장부 URL 쿼리 (공통)

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `tab` | `purchase` \| `sale` \| `income` \| `products` | 메인 탭 |
| `purchaseSub` | `product` \| `supply` \| `other` | 매입 서브탭 (매입일 때) |
| `month` | `YYYY-MM` | 년도·월 탭 선택 (상품관리 제외) |
| `q` | 문자열 | **탭 내 목록 검색어** (아래 전역 검색과 동일 키) |

- 탭 전환·월 변경·검색어 입력 시 `router.replace`로 URL 갱신 (뒤로가기 가능)

### 장부 전역 검색 (구현됨, 퍼블)

탭 내 검색(해당 탭 목록만 필터)과 별도로, **장부 전체**에서 키워드를 찾는 검색을 제공한다.

**진입**

- 장부 헤더 **년도 선택 행** 우측 `Search` 아이콘 → 모달 오픈
- 년도(◀ 2026년 ▶)는 가운데 고정, 검색 버튼만 우측

**모달 동작**

- 입력 즉시 결과 목록 갱신 (클라이언트 검색)
- 결과 한 줄: **탭 뱃지** (`매입 · 상품매입` 등) + **날짜**(있을 때) + **제목** + **부제**
- 항목 클릭 시 모달 닫고 아래 URL로 이동

**결과 클릭 시 딥링크 (자동 연동)**

1. `tab` / `purchaseSub`(매입) / `month`(해당 내역 날짜의 `YYYY-MM`) / `q`(검색어) 설정
2. 해당 메인·서브탭 활성화
3. 월 탭 URL 반영 (상품관리는 `month` 생략 가능)
4. 탭 본문 검색 툴바에 `q` 반영 → 기존 탭 내 필터 로직으로 목록 자동 축소

예시:

```
/ledger?tab=purchase&purchaseSub=product&month=2026-06&q=피규어
/ledger?tab=sale&month=2026-06&q=김민수
/ledger?tab=products&q=SKU-001
```

**검색 대상 필드 (탭 내 검색과 동일 기준)**

| 구분 | 검색 필드 |
|------|-----------|
| 상품매입 | 그룹명, 상품명, 구매처, 주문번호, 비고, 결제날짜 |
| 부가 | 항목명, 구매처, 비고, 결제날짜 |
| 기타지출 | 항목명, 비고, 결제날짜 |
| 매출 | 주문번호, 주문자명, 주문항목 상품명, 비고, 주문일 |
| 수익 | 항목명, 비고, 입금일 |
| 상품관리 | 상품명, SKU, 메모, 카테고리 |

**퍼블 데이터 소스 (현재)**

- 전역 검색 인덱스: 매입·매출·수익 **시드 데이터** + 상품관리 **localStorage** (`sellog-products-pub-v1`)
- 탭에서 새로 등록·수정한 매입/매출/수익(컴포넌트 state)은 **전역 검색에 즉시 반영되지 않을 수 있음**
- 상품관리에서 등록·수정한 상품은 localStorage 기준으로 반영

**구현 파일**

- `src/components/ledger/ledger-global-search-dialog.tsx`
- `src/lib/ledger-global-search.ts`
- `src/hooks/use-ledger-url-search.ts`
- `src/components/ledger/ledger-year-picker.tsx` (검색 트리거)

**추후 (BE·완성도)**

- 서버 통합 검색 API (`GET /ledger/search?q=`) 및 페이징
- 장부 데이터 공유 저장소/API 연동 후 세션 등록분도 전역 검색 반영
- `focus` 또는 `lineId` 파라미터로 **특정 행** 스크롤·하이라이트
- 월 선택과 목록 필터 연동 후, 딥링크 `month`와 화면 목록 일치

## 손익 계산 정책 (현행 합의)

- 기타지출은 수익 탭 내부 계산에서 분리
- 장부 상단 추이에서 `수익(입금) - 누적 기타지출 = 총` 형태로 별도 표시
- 상단 추이 카드(매입·매출·수익)는 **퍼블 시드 데이터 기준 전체 합산값** 표시
  - BE 연동 시 API 집계 응답으로 교체

## 매입

### 서브탭

- `상품매입`
- `부가`
- `기타지출`

### 공통 동작

- 결제날짜 기준 자동 그룹핑(날짜별 묶음)
- 그룹 펼침/접기
- 각 그룹 하단 요약(총 주문금액, 추가금, 할인금액, 총 지출)
- 검색 + 등록 버튼 툴바 제공 (`LedgerListShell` — 검색·목록 한 카드)
- 탭 내 검색어는 URL `q`와 동기화 (`useLedgerUrlSearch`)

### 출금계좌 (상품매입·부가·기타 공통)

- 카테고리 패턴: `GET/POST/PATCH/DELETE /banks`로 계좌 등록 후 매입 등록·수정 시 선택
- 등록 모달: **출금계좌** 행 + 「선택」→ `bank-manage-dialog` (은행명·계좌번호·예금주)
- 선택 라벨: `카카오뱅크 · 3333-01-1234567 · 홍길동`
- API body: `bankId` (선택, 생략 시 null)
- 목록·상세 응답: `bankId`, `bank` 객체 (삭제된 계좌는 `bank: null` 가능)

### 상품매입

입력 항목(현재 구현 기준):

- 결제날짜
- 주문번호(선택)
- 상품이미지(선택, 퍼블은 브라우저 data URL)
- 상품명
- 상품상세링크(선택)
- 구매처
- 수량
- 결제금액
- **출금계좌** (선택, 관리 모달에서 등록·선택)
- 비고(선택)

라인 데이터 구조:

| 필드 | 타입 | 설명 |
|------|------|------|
| `productSku` | `string?` | 재고반영 시 선택된 상품관리 SKU (확정 전 미설정) |
| `stockReflected` | `boolean` | 재고반영 여부 |

가격 정책(합의):

- 재고반영 확정 시 `결제금액 ÷ 반영수량 = 단가` 계산
- 해당 단가가 상품의 현재 기본가격과 다를 때만 자동 갱신
- 상품 기본가격 변경은 전부 `priceHistory`에 누적 저장(상품관리 수동수정 포함)

계산/표시:

- 개당금액 자동계산
- 그룹 추가금/할인 반영한 최종개당금액 계산
- 추천판매가 범위 계산(퍼블 마진율 15~50%)

그룹 액션:

- 일괄 재고반영 버튼 (미반영 라인 순서대로 모달 호출)
- 일괄 주문취소 버튼 (퍼블 상태 처리 중심)

삭제 제약:

- `stockReflected: true` 라인은 삭제 불가 (반영취소 후 삭제 가능)

### 부가

입력 항목(현재 구현 기준):

- 결제날짜
- 항목명
- 구매처(선택)
- 수량
- 금액
- 비고(선택)

- 라인별 재고반영 가능
- 재고반영 시 재고반영 모달 사용(상품매입과 동일 모달)

### 기타지출

입력 항목(현재 구현 기준):

- 결제날짜
- 항목명
- 금액
- 비고(선택)

- 재고반영 없음
- 날짜 기준 그룹형 목록

## 재고반영 모달 (상품매입/부가 공용)

현재 구현 UI/동작:

- 제목: `{대상 상품명} 재고 반영`
- **상품 선택**: 상품관리 `GET /products` 목록에서 선택 (`active` 상품)
  - input focus 시 드롭다운 오픈, 실시간 SKU·상품명 검색
  - **드롭다운 최상단 `+ 상품 추가`**: 상품관리 등록 모달을 같은 흐름에서 오픈 (탭 이동 없음)
  - 등록된 상품이 0개여도 검색 input·드롭다운 표시 (최상단 `+ 상품 추가`만으로도 진입 가능)
  - 선택 후 "현재 재고 N개 → 반영 후 M개" 미리보기 표시
- **신규 상품 등록 (재고반영 경로)**:
  - 매입 라인 정보를 상품 등록 폼에 **prefill** (수정 가능)
  - 매핑: `productName`/`itemName` → `name`, `imageUrl`(http URL), `paymentAmount÷quantity` → `currentPrice`, 구매처·주문번호·링크·memo → `memo` 보조
  - **초기재고(`stock`)는 0 고정** — 실제 재고 증가는 재고반영 확정 API로만 처리 (이중 반영 방지)
  - 등록 성공 시 해당 SKU 자동 선택 → 재고반영 모달로 복귀
- **반영 수량**: 증감 버튼(+/-) 및 직접 입력 (기본값 = 구매 수량)
- **확정 시 처리 (상품매입)**:
  1. 상품 마스터 `stock` 증가
  2. `stockHistory` 항목 추가 (출처: `purchase`, 구매처·주문번호·단가·금액 포함)
  3. 반영 단가 ≠ 현재 기본가격일 때 `currentPrice` 갱신 + `priceHistory` 추가
  4. 매입 라인 `stockReflected: true`, `productSku` 저장
- **확정 시 처리 (부가)**:
  1. 상품 마스터 `stock` 증가
  2. `stockHistory` 항목 추가 (출처: `purchase`, reason: 공급비용 재고반영)
  3. 가격 업데이트는 미적용 (공급품·소모품 특성)
  4. 부가 라인 `stockReflected: true` 저장

**SKU 기반 연결 정책**

- `ProductPurchaseLine.productSku` — 재고반영 시 선택한 상품의 SKU 저장
- SKU = 상품관리의 고유 상품 ID (BE 연동 기준)
- 매입 라인 ↔ 상품 조회는 `sku` 필드로 매칭

**재고반영 취소**

- `stockReflected: true` 상태에서 "반영취소" 선택 시 `stockReflected: false`만 변경
- **상품 마스터 stock 복구는 수동 재고조정으로 처리** (퍼블 제약)
- BE 연동 시: 반영취소 API 호출 → stock 차감 + stockHistory 역분개

## 상품관리

### 전체 구조

- 월 필터 무관 (전체 기준)
- 좌측 상품 목록 + 우측 상세 패널 2단 레이아웃
- 초기 퍼블 샘플 1건 존재(`SKU-001`)

### 좌측 상품 목록

- 검색: 상품명 / SKU / 메모 (`q` URL 연동)
- 상단 툴바: 검색 + `+ 상품 등록`
- 카드형 목록
  - 썸네일, 상품명, SKU, 안전재고, 현재재고, 상태 뱃지(정상/안전재고 이하/비활성)

### 상품 등록/편집 모달

입력 항목:

- SKU(수동 입력)
- 상품명
- 카테고리(읽기 전용 표시 + `선택` 버튼으로 카테고리 모달 오픈)
- 초기재고
- 안전재고
- 기본 가격(필수)
- 사용여부(활성/비활성)
- 상품이미지(업로드/삭제)
- 비고

삭제 정책:

- 현재 재고가 0개일 때만 삭제 가능
- 삭제는 hard delete가 아니라 `deletedAtIso` soft delete 처리

### 카테고리 관리 모달

- 기능: 선택 / 추가 / 수정 / 삭제(soft delete)
- 등록/편집 모달에서 호출하여 카테고리 반영

### 우측 상품 상세

- 기본 정보: 이미지, 상품명, SKU, 카테고리
- 요약 카드: 현재재고 / 안전재고 / 사용여부 / 기본가격
- 재고조정 버튼:
  - 증가/감소 선택
  - 수량 입력
  - 사유(선택)
  - 적용 후 재고 이력 기록
- **재고 이력 목록** 표시(최신순)
  - source: `purchase`(매입/공급비용 반영) / `sale`(매출 차감) / `manual_adjust`(수동조정)
  - 매입 반영: 구매처·단가·금액 포함
  - 매출 차감: 주문번호 포함 → **BE 연동 시 판매채널도 포함 예정**
- **가격 변경 이력 목록** 표시(최신순)
  - source: `product_register`(등록 시) / `manual_edit`(수동수정) / `purchase`(매입 반영)

가격 기본값 규칙:

- 매출 등록의 주문항목에서 상품 선택 시, 기본 금액은 해당 상품의 **최신 기본가격** 자동 적용
- 상품관리에서 가격 수동 수정 시 즉시 최신값 갱신 + 가격 이력 기록
- 매입 재고반영 확정 시 단가 변경 여부에 따라 자동 갱신 + 이력 기록

- 상단 요약: 오늘 매출 / 이번달 매출
- 등록 모달:
  - 주문일 / 주문번호 / 주문자명 / **판매채널** / 주문항목(다건) / 추가금액(라벨+금액 다건) / 할인금액(라벨+금액 다건) / 비고
  - 총 주문금액 자동 계산: `주문항목 합 + 추가금액 합 - 할인금액 합`
  - 주문항목 상품 선택 시 상품관리 기본가격 자동 반영
- 목록:
  - 검색(주문번호/주문자명/상품명/비고, URL `q` 연동)
  - 상태(정상/취소), 수정/취소처리/취소해제/삭제
  - 채널 뱃지 표시 (등록 시 선택 시)
- 재고 연동:
  - 저장 시 주문항목 수량만큼 차감 (재고 부족 시 저장 차단)
  - **취소처리** 시 즉시 재고 복구
  - **취소해제** 시 재고 재차감 (재고 부족이면 취소해제 차단)
  - **수정** 시: 기존 차감 복구 → 재고 재검증 → 새 수량 차감
  - **삭제**: 반드시 취소 상태에서만 가능 (재고는 취소처리 시 이미 복구됨)

## 수익 탭

현재 상태: **MVP 구현 (입금 기록)**

- 상단 요약: 오늘 입금 / 이번달 입금
- 입금 등록 모달 필드:

| 필드 | 필수 | 설명 |
|------|------|------|
| 입금일 | ✓ | |
| 항목명 | ✓ | 예: 네이버 정산, 쿠팡 정산 |
| 금액 | ✓ | 실제 입금된 금액 (gross) |
| 부가세 | — | 정보용, 금액 계산에 직접 사용 안 함 |
| 플랫폼 수수료 | — | 정보용, 금액 계산에 직접 사용 안 함 |
| 비고 | — | |

- 부가세·수수료 활용 방향 (BE 연동 시):
  - 순수익 계산: `순수익 = 금액 - 부가세 - 수수료`
  - 현재는 입력만 가능, UI 집계 미반영
- 목록: 입금일 기준 그룹, 검색(항목명/비고, URL `q` 연동), 수정/삭제
  - 부가세·수수료 컬럼 표시 (값 없으면 `—`)
- 상단 `+ 입금 등록`만 제공
