## 공통

**GNB:** 대시보드 / 장부 / **소싱** / 설정  
**장부 탭:** 매입 / 매출 / 수익 / 상품관리

### 장부 헤더 (정산 추이)

- 상단 `정산 추이` 영역은 접기/펼치기 가능
- 프리셋 버튼: `전체 / 올해 / 이번달 / 주간 / 오늘` → **`GET /ledger/summary?period=` 연동**
- **카드 3개 표시: `매입 / 매출(정상) / 수익(입금)`**

**정산 추이 계산 기준** (`BE-guide.md` §1과 동일)

| 카드 | 계산 공식 | 날짜 기준 |
|------|-----------|-----------|
| 매입 | 상품매입 + 부가 (기타지출 제외) | `payment_date` |
| 매출 (정상) | `status = "normal"` 주문 `totalAmount` 합 | `order_date` |
| 수익 (입금) | 입금 `amount` 합 | `deposit_date` |

- 추이 하단 요약:
  - `수익 입금 N원` (선택 기간)
  - `- 누적 기타지출 M원` (기타지출 **전체 누적**, period 무관)
  - `= 총 K원` (`netTotal`)
- 구현: `useLedgerSummary` + `ledger-header.tsx`
- 매입·매출·수익 등록·수정·삭제 시 summary 쿼리 invalidate

### 월 선택 표시 정책

- `매입/매출/수익` 탭: 월 선택 버튼 표시 → **URL `month=YYYY-MM` + 목록 API `month` 파라미터 연동**
- `상품관리` 탭: 월 선택 버튼 숨김 (엑셀만 전체/활성)
- 월 탭 범위: `GET /ledger/earliest-month` (년도·탭별 최초 데이터 월)
- 월 탭 우측: **엑셀 다운로드** 버튼 (아래 §엑셀 참조)

### 장부 URL 쿼리 (공통)

| 파라미터 | 값 | 설명 |
|----------|-----|------|
| `tab` | `purchase` \| `sale` \| `income` \| `products` | 메인 탭 |
| `purchaseSub` | `product` \| `supply` \| `other` | 매입 서브탭 (매입일 때) |
| `month` | `YYYY-MM` | 년도·월 탭 선택 (상품관리 제외) |
| `q` | 문자열 | **탭 내 목록 검색어** (아래 전역 검색과 동일 키) |

- 탭 전환·월 변경·검색어 입력 시 `router.replace`로 URL 갱신 (뒤로가기 가능)

### 장부 전역 검색 (구현됨, 퍼블)

탭 내 검색(해당 탭 목록만 필터)과 별도로, **장부 전체**에서 키워드를 찾는 검색을 제공한다.

**진입**

- 장부 헤더 **년도 선택 행** 우측 `Search` 아이콘 → 모달 오픈
- 년도(◀ 2026년 ▶)는 가운데 고정, 검색 버튼만 우측

**모달 동작**

- 입력 즉시 결과 목록 갱신 (클라이언트 검색)
- 결과 한 줄: **탭 뱃지** (`매입 · 상품매입` 등) + **날짜**(있을 때) + **제목** + **부제**
- 항목 클릭 시 모달 닫고 아래 URL로 이동

**결과 클릭 시 딥링크 (자동 연동)**

1. `tab` / `purchaseSub`(매입) / `month`(해당 내역 날짜의 `YYYY-MM`) / `q`(검색어) 설정
2. 해당 메인·서브탭 활성화
3. 월 탭 URL 반영 (상품관리는 `month` 생략 가능)
4. 탭 본문 검색 툴바에 `q` 반영 → 기존 탭 내 필터 로직으로 목록 자동 축소

예시:

```
/ledger?tab=purchase&purchaseSub=product&month=2026-06&q=피규어
/ledger?tab=sale&month=2026-06&q=김민수
/ledger?tab=products&q=SKU-001
```

**검색 대상 필드 (탭 내 검색과 동일 기준)**

| 구분 | 검색 필드 |
|------|-----------|
| 상품매입 | 그룹명, 상품명, 구매처, 주문번호, 비고, 결제날짜 |
| 부가 | 항목명, 구매처, 비고, 결제날짜 |
| 기타지출 | 항목명, 비고, 결제날짜 |
| 매출 | 주문번호, 주문자명, 주문항목 상품명, 비고, 주문일 |
| 수익 | 항목명, 비고, 입금일 |
| 상품관리 | 상품명, SKU, 메모, 카테고리 |

**퍼블 데이터 소스 (현재)**

- 전역 검색 인덱스: 매입·매출·수익 **시드 데이터** + 상품관리 **API/localStorage 혼용**
- 탭 목록은 BE API 기준이나, 전역 검색은 **`GET /ledger/search` 미연동** → 세션 등록분 즉시 반영 안 될 수 있음

**구현 파일**

- `src/components/ledger/ledger-global-search-dialog.tsx`
- `src/lib/ledger-global-search.ts`
- `src/hooks/use-ledger-url-search.ts`
- `src/components/ledger/ledger-year-picker.tsx` (검색 트리거)

**추후 (BE·완성도)**

- 서버 통합 검색 API (`GET /ledger/search?q=`) 및 페이징
- 장부 데이터 공유 저장소/API 연동 후 세션 등록분도 전역 검색 반영
- `focus` 또는 `lineId` 파라미터로 **특정 행** 스크롤·하이라이트
- 월 선택과 목록 필터 연동 후, 딥링크 `month`와 화면 목록 일치

## 손익 계산 정책 (현행 합의)

- 기타지출은 수익 탭 내부 계산에서 분리
- 장부 상단 추이에서 `수익(입금) - 누적 기타지출 = 총` 형태로 별도 표시
- **매출 추정 순익** (`marginEstimate`): BE가 주문별 계산 → FE 표시만
  - 추정 순수익(초록/빨강) + 괄호 안 추정 실수익(부가세·수수료 반영)
  - `hasUnknownCost` 시 `※ 일부 품목 원가 미확인`
  - 취소 주문은 `marginEstimate: null` → 영역 숨김
  - 등록·수정 모달: `POST /sales/estimate-margin` 디바운스 미리보기
- 추천 판매가·순익 가정값 (설정 연동 전 하드코딩):
  - 마진 15~50% (`PUB_MARGIN_MIN` / `PUB_MARGIN_MAX`)
  - 부가세 `× 10/110`, 플랫폼 수수료 `× 6.36%`
  - → **설정 화면·API 연동 후 사용자 값으로 교체 예정** (`BE-guide.md` §14-2)

## 매입

### 서브탭

- `상품매입`
- `부가`
- `기타지출`

### 공통 동작

- 결제날짜 기준 자동 그룹핑(날짜별 묶음)
- 그룹 펼침/접기
- 각 그룹 하단 요약(총 주문금액, 추가금, 할인금액, 총 지출)
- 검색 + 등록 버튼 툴바 제공 (`LedgerListShell` — 검색·목록 한 카드)
- 탭 내 검색어는 URL `q`와 동기화 (`useLedgerUrlSearch`)

### 출금계좌 (상품매입·부가·기타 공통)

- `GET/POST/PATCH/DELETE /banks` — **BE 연동 완료**
- 등록 모달: **출금계좌** 행 + 「선택」→ `bank-manage-dialog`
- API body: `bankId` (선택, 생략 시 null)

### 구매처 (상품매입·부가)

- `GET/POST/PATCH/DELETE /vendors` — **BE 연동 완료** (`VENDORS_USE_API = true`)
- 등록 모달: 구매처 **선택** + 관리 모달 (`vendor-manage-dialog`)
- 매입 저장 시 `vendorId` 전송, 응답에 `vendorSnapshot` (`{ id, name, link }`)
- 부가: 구매처 선택 (선택 필드)

### 상품매입

입력 항목:

- 결제날짜, 주문번호(선택), 상품이미지(선택), 상품명, 상품상세링크(선택)
- **구매처** (관리 모달에서 선택, `vendorId` 필수)
- 수량, 결제금액, 출금계좌(선택), 비고(선택)

**목록 그룹 구조 (2단)**

```
결제일 그룹
  └─ 구매처 그룹 (vendorGroups)
       ├─ 라인 목록
       ├─ 구매처별 추가금·할인·소계
       └─ 구매처별 [재고반영] / [주문취소]
  └─ 결제일 하단 합계 (totals)
```

- 구매처별 추가금·할인: `PATCH /purchase/groups/vendor`
- 구매처별 주문취소: `PATCH /purchase/groups/vendor/cancel`

라인 데이터 구조:

| 필드 | 타입 | 설명 |
|------|------|------|
| `productSku` | `string?` | 재고반영 시 선택된 상품관리 SKU (확정 전 미설정) |
| `stockReflected` | `boolean` | 재고반영 여부 |

가격 정책(합의):

- 재고반영 확정 시 `결제금액 ÷ 반영수량 = 단가` 계산
- 해당 단가가 상품의 현재 기본가격과 다를 때만 자동 갱신
- 상품 기본가격 변경은 전부 `priceHistory`에 누적 저장(상품관리 수동수정 포함)

계산/표시:

- 개당금액, 최종개당금액, **추천판매가 범위**(마진 15~50%, 우측 정렬)
- **재고 미반영** 라인: 행 배경 `bg-red-50`
- 재고반영 후: 「반영완료」+ 「반영취소」(11px bold 빨강) 세로 가운데 정렬

그룹 액션:

- 결제일 단위: 미반영 일괄 재고반영
- **구매처 단위**: 미반영 일괄 재고반영, 구매처별 주문취소

삭제 제약:

- `stockReflected: true` 라인은 삭제 불가 (반영취소 후 삭제 가능)

## 재고반영 모달 (상품매입/부가 공용)

현재 구현 UI/동작:

- 제목: `{대상 상품명} 재고 반영`
- **상품 선택**: 상품관리 `GET /products` 목록에서 선택 (`active` 상품)
  - input focus 시 드롭다운 오픈, 실시간 SKU·상품명 검색
  - **드롭다운 최상단 `+ 상품 추가`**: 상품관리 등록 모달을 같은 흐름에서 오픈 (탭 이동 없음)
  - 등록된 상품이 0개여도 검색 input·드롭다운 표시 (최상단 `+ 상품 추가`만으로도 진입 가능)
  - 선택 후 "현재 재고 N개 → 반영 후 M개" 미리보기 표시
- **신규 상품 등록 (재고반영 경로)**:
  - 매입 라인 정보 prefill (`purchase-to-product-prefill.ts`)
  - **판매가** 라벨 옆 **추천 판매가 범위** (목록과 동일 계산)
  - 초기재고(`stock`) 0 고정 — 재고 증가는 재고반영 API만
- **반영 수량**: 증감 버튼(+/-) 및 직접 입력 (기본값 = 구매 수량)
- **확정 시 처리 (상품매입)**:
  1. 상품 마스터 `stock` 증가
  2. `stockHistory` 항목 추가 (출처: `purchase`, 구매처·주문번호·단가·금액 포함)
  3. 반영 단가 ≠ 현재 기본가격일 때 `currentPrice` 갱신 + `priceHistory` 추가
  4. 매입 라인 `stockReflected: true`, `productSku` 저장
- **확정 시 처리 (부가)**:
  1. 상품 마스터 `stock` 증가
  2. `stockHistory` 항목 추가 (출처: `purchase`, reason: 공급비용 재고반영)
  3. 가격 업데이트는 미적용 (공급품·소모품 특성)
  4. 부가 라인 `stockReflected: true` 저장

**SKU 기반 연결 정책**

- `ProductPurchaseLine.productSku` — 재고반영 시 선택한 상품의 SKU 저장
- SKU = 상품관리의 고유 상품 ID (BE 연동 기준)
- 매입 라인 ↔ 상품 조회는 `sku` 필드로 매칭

**재고반영 취소**

- BE: `DELETE /purchase/products/:id/stock-reflect` — stock 차감 + stockHistory 역분개
- 이미 매출로 차감된 수량이 있으면 `409` (반영취소 불가)

## 상품관리

### 전체 구조

- 월 필터 무관 (전체 기준)
- 좌측 상품 목록 + 우측 상세 패널 2단 레이아웃
- 초기 퍼블 샘플 1건 존재(`SKU-001`)

### 좌측 상품 목록

- 검색: 상품명 / SKU / 메모 (`q` URL 연동)
- 상단 툴바: 검색 + `+ 상품 등록`
- 카드형 목록: 썸네일, 상품명, SKU, 안전재고, 현재재고, 상태 뱃지(정상/품절임박/품절/비활성)
- **재고 상태 필터** (품절·품절임박만 보기): **미구현** → `GET /products?stockStatus=` 연동 예정

### 상품 등록/편집 모달

입력 항목:

- SKU, 상품명, 카테고리, 초기재고, 안전재고
- **판매가** (필수, API 필드 `currentPrice`)
- 사용여부, 상품이미지, 비고

삭제 정책:

- 현재 재고가 0개일 때만 삭제 가능
- 삭제는 hard delete가 아니라 `deletedAtIso` soft delete 처리

### 카테고리 관리 모달

- 기능: 선택 / 추가 / 수정 / 삭제(soft delete)
- 등록/편집 모달에서 호출하여 카테고리 반영

### 우측 상품 상세

- 요약 카드: 현재재고 / 안전재고 / 사용여부 / **판매가**
- 재고조정 버튼:
  - 증가/감소 선택
  - 수량 입력
  - 사유(선택)
  - 적용 후 재고 이력 기록
- **재고 이력 목록** 표시(최신순)
  - source: `purchase`(매입/공급비용 반영) / `sale`(매출 차감) / `manual_adjust`(수동조정)
  - 매입 반영: 구매처·단가·금액 포함
  - 매출 차감: 주문번호 포함 → **BE 연동 시 판매채널도 포함 예정**
- **가격 변경 이력 목록** 표시(최신순)
  - source: `product_register`(등록 시) / `manual_edit`(수동수정) / `purchase`(매입 반영)

가격 기본값 규칙:

- 매출 등록의 주문항목에서 상품 선택 시 **판매가(`currentPrice`)** 자동 적용
- 상품관리에서 가격 수동 수정 시 즉시 최신값 갱신 + 가격 이력 기록
- 매입 재고반영 확정 시 단가 변경 여부에 따라 자동 갱신 + 이력 기록

## 소싱 `/sourcing` (기획 · UI 퍼블 예정)

> **구현 순서:** 서비스 기획(본 절) → **UI 퍼블리싱** → **BE 작업** (`BE-guide.md` §소싱) → **FE API 연동**  
> 현재: UI 퍼블(in-memory) · BE·export API 미연동

### 목적

- **즐겨찾기·메모용** 소싱 데이터 — 눈여겨보는 도매몰·후보 상품을 모아 두는 공간
- **장부·상품관리·구매처·판매채널과 연관 없음** (FK·동기화·재고/손익 반영 없음)
- 소싱 도메인 **내부**에서만 연결: 제품소싱 → (선택) 소싱채널
- 향후 「매입 등록 prefill」 등은 **수동 편의**일 뿐, 데이터 자동 연동 아님

### 페이지 구조

```
/sourcing?tab=channels|products
```

| URL 파라미터 | 값 | 설명 |
|--------------|-----|------|
| `tab` | `channels` \| `products` | 메인 탭 (기본: `products` 또는 `channels` — 퍼블 시 확정) |
| `q` | 문자열 | 탭 내 검색어 |
| `page` | 숫자 | 목록 페이지 (1부터) |

- 상단: 페이지 제목 `소싱`
- 탭 2개: **채널소싱** / **제품소싱** (마스터 → 상세 흐름상 **채널소싱을 왼쪽** 배치 권장)
- 각 탭: 검색 툴바 + 등록 버튼 + 목록 + **페이징**
- 검색: Enter 또는 검색 버튼 (매입 목록과 동일, debounce 없음)

### 탭 1 — 채널소싱

**역할:** 소싱처(도매몰·1688·공장 등) 마스터

| 필드 | 필수 | 설명 |
|------|------|------|
| 상호명 | ✓ | 채널 표시명 |
| URL | — | 쇼핑몰·카탈로그 링크. 목록·상세에서 **`바로가기`** 로 표시, `target="_blank"` |

**CRUD:** 등록 · 목록 · 수정 · 삭제

**중복 정책 (BE 검증):**

- 동일 사용자 기준 **`상호명` + `URL` 조합**이 이미 있으면 신규 등록 거부
- URL 정규화 후 비교 (BE — `BE-guide.md` §소싱-채널)
- FE: 409 등 에러 메시지를 공통 alert 모달로 표시

**삭제 정책:**

- 해당 채널에 **연결된 제품소싱이 1건이라도 있으면 삭제 불가**
- FE: 경고 **모달** — 안내 문구 + 연결된 **상품 목록** 표시
- 연결 제품을 **모두 삭제**(또는 다른 채널로 변경)한 뒤에만 채널 삭제 가능
- BE: `409 CHANNEL_HAS_PRODUCTS` + 연결 제품 요약 (아래 `BE-guide.md`)

**목록 UI (퍼블):**

- 테이블: 상호명 · 바로가기 · (선택) 연결 제품 수 · 수정/삭제
- 검색 대상: **상호명**, **URL** 일부

### 탭 2 — 제품소싱

**역할:** 후보 상품 카드/행

| 필드 | 필수 | 설명 |
|------|------|------|
| 소싱채널 | — | 채널소싱에서 선택. **미선택 가능** |
| 이미지 | — | 기존 `ImageUploadModal` + `POST /upload` 재사용 |
| 상품명 | ✓ | |
| 가격 정보 | ✓ | 아래 **가격 입력 정책** |
| URL | — | 상품 상세 링크 · **`바로가기`** (새 창) |
| 메모 | — | 가격·MOQ·통화 등 자유 메모 (원화 외 안내 등) |

**소싱채널 선택 UX:**

- 구매처·판매채널과 동일: 필드 클릭 → **채널 관리 모달** (선택 · 등록 · 수정 · 삭제 · 선택 해제)
- **제품 등록 모달 안에서 채널 즉시 추가** 가능 (채널 탭 이동 없이) — 등록 성공 시 해당 채널 자동 선택

**가격 입력·표시 정책:**

- 사용자 **직접 입력** (자동 계산 필드 아님)
- 입력 항목:
  - **수량** (개, 1 이상 정수)
  - **개별 단가** (원, KRW 기준)
- **표시:**
  - `개별 N원` (또는 `개당 N원`)
  - `전체 M원` = 수량 × 개별 단가 (FE에서 표시용 계산, 저장값도 quantity·unitPrice로 분리)
- **메모:** 별도 textarea (예: MOQ, 배송비, 달러 환산 참고 — **원화 기준이 기본**, 메모에 통화·조건 기록)

**CRUD:** 등록 · 목록 · 수정 · 삭제

**목록 UI (퍼블):**

- **카드 그리드** 권장: 썸네일 · 상품명 · 채널명(없으면 `-`) · 개별/전체 금액 · 바로가기
- 검색 대상: **상품명**, **가격(수량·단가·합계 문자열)**, **연결 채널 상호명**

### 엑셀 다운로드

> 장부 엑셀과 동일 UX: 탭별 툴바 **「엑셀다운로드」** 버튼 · 전체 등록 데이터 기준 (검색·페이징 무관)
