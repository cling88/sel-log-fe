### 장부 공통 레이아웃

```
┌─────────────────────────────────────────────┐
│  [전체] [올해] [이번달] [주간] [오늘]         │  ← 추이 토글 (장부 전체 고정)
├─────────────────────────────────────────────┤
│  [년도 ▼]  ← 올해 선택 시                   │
│  [월 ▼]    ← 이번달 선택 시                 │
│  [주 ▼]    ← 주간 선택 시                   │
│  (전체/오늘은 select 없음)                   │
├─────────────────────────────────────────────┤
│  [매입 X원]  [매출 X원]  [수익 X원]          │
│  [매입 X건]  [매출 X건]  [순수익 X원]        │
│                         이전기간 대비 ▲▼     │
├─────────────────────────────────────────────┤
│  [2026년 5월 ▼]  ← 매입/매출/수익 탭일 때만 표시
│                    상품관리 탭 클릭 시 숨김   │
├─────────────────────────────────────────────┤
│  [ 매입 ] [ 매출 ] [ 수익 ] [ 상품관리 ]     │
└─────────────────────────────────────────────┘
```

- 추이 토글: 장부 페이지 전체 고정 (탭 전환과 무관하게 항상 표시)
- 월 선택: 매입/매출/수익 탭일 때만 표시. 상품관리 탭 클릭 시 숨김
- 상품관리: 월 선택 무관, 전체 데이터 기준
- 이전기간 대비: 전체 탭 제외 나머지 4개에만 표시

매출총이익  = 매출 - 매입원가        (장부 기준)

### 매입 서브탭

| 서브탭 | 용도 |
|--------|------|
| **상품매입** | 판매 상품 구매, 그룹·재고반영·원가·추천판매가 |
| **부가** | 포장재·소모품 등 (재고 반영 여부 항목별 선택) |
| **기타지출** | 월세·촬영·연구비·세무·통신 등 — 날짜/항목명/금액/비고 단순 목록 |

### 상품매입 — 그룹 개념 (자동 묶음)

**그룹은 사용자가 따로 만드는 단위가 아닙니다.** 목록에 보이는 묶음일 뿐입니다.

- 등록: **매입 라인(상품 1건)을 날짜별로 하나씩** 등록 (`+ 상품 매입 등록하기` 등)
- 묶음: **같은 결제날짜**의 라인은 시스템이 **자동으로 한 그룹**으로 표시 (펼치기/접기)
- 그룹명: 미지정 시 `매입N` 또는 `YYYY-MM-DD` 등 표시 규칙 (구현 시 확정). 그룹 헤더에서만 수정 가능
- 그룹 단위 **추가금·할인**은 그 날짜 묶음에만 적용 (배송비·쿠폰을 하루 주문에 한 번)

```
[등록 UX]  라인 1건 입력 (결제날짜 포함) → 저장
[목록 UX]  결제날짜별 그룹 카드 → 펼치면 그날 라인들 + 하단 요약
```

- 페이지네이션: **그룹 단위**
- 추후 세금계산서 제출용 엑셀 다운로드 가능
- 정해진 양식에 맞게 엑셀 업로드 시 자동 등록 (동일 날짜·그룹번호 규칙으로 묶음)

---

그룹을 먼저 만들지 않습니다. 아래를 입력해 저장하면 해당 **결제날짜 그룹**에 들어갑니다.

| 상품이미지 | △ | 파일 업로드(선택), 퍼블은 data URL 로컬 저장 |
| 상품명 | ○ | |
| 상품상세링크 | △ | |
| 구매처 | ○ | |
| 개수 | ○ | |
| 결제금액 | ○ | |
| 비고 | △ | |

자동: 개당금액, (그룹 요약 반영 후) 최종개당금액·추천판매가  
재고반영: 등록 직후가 아니라 라인별 **재고반영** 버튼·모달 (후술)

---

### 부가 — 등록 시 입력 (라인 1건)

같은 **결제날짜**끼리 자동 묶음 (상품매입과 동일 UX). 재고는 **상품관리** 마스터와 공통.

| 항목 | 필수 | 비고 |
|------|------|------|
| 결제날짜 | ○ | 기본값 오늘 |
| 항목명 | ○ | 포장·소모품 등 |
| 구매처 | △ | |
| 수량 | ○ | 기본 1 |
| 금액 | ○ | |
| 비고 | △ | textarea |

재고반영: 라인별 선택. 미반영 시 당월 비용만 집계. 반영 시 상품관리 SKU 선택 모달(상품매입과 동일 구조).

### 엑셀 업로드/다운로드

**업로드**
- 테이블 상단 우측: [샘플다운로드] [엑셀업로드] [엑셀다운로드]
- 샘플 양식 컬럼: 추후 확정 예정 (그룹번호 기준으로 동일 그룹번호끼리 묶어서 1개 그룹으로 처리)
  - 그룹번호는 로직에서만 참조, 예: 001
  - 카테고리/SKU 없으면 자동 생성
- 업로드 후 미리보기 테이블 표시 (오류 행 빨간색 표시)

**다운로드**

### 그룹 단위 목록 (표시 전용)

: 번호 / 결제날짜 / 그룹명 / 총 결제금액 / 수정버튼 / 삭제버튼

- 그룹 = **같은 결제날짜** 라인의 자동 묶음 (등록 화면에서 「그룹 만들기」 없음)
- 그룹명: 헤더에서 수정 가능. 기본값 `매입N` 또는 날짜 라벨 (구현 시 확정)
- 수정 클릭 시: 그룹 헤더의 결제날짜·그룹명만 수정 (해당 날짜 묶음 전체에 적용)
- 삭제 클릭 시: 그룹 전체 완전 삭제
  - 재고반영된 목록이 1건이라도 있으면 삭제 불가 → 전체 반영취소 후 삭제 가능
  - 주문취소 상태인 그룹: 삭제 불가. 단, 완전삭제(강제삭제) 옵션 선택 시 삭제 가능
- 주문취소 상태인 그룹: 삭제버튼 / 주문취소해제버튼 / 드래그 순서이동만 가능. 나머지 기능 전부 비활성화
- 그룹 hover 시: 테이블 바깥 좌측에 draggable 아이콘 표시 → 그룹 순서 변경 가능
- 재고반영이 필요한 목록이 있는 그룹: 그룹 배경색 다르게 표시

---

### 목록 컬럼

: 표시정보가 많아 테이블보다 카드 목록 형태로 구성 (한눈에 들어올 수 있도록)
: 번호 / 주문번호(선택입력) / 상품이미지 / 상품명 / 상품상세링크 / 구매처 / 개수 / 결제금액 / 개당금액 / 최종개당금액 / 추천판매가 / 상태 / 비고 / 3dots

**자동계산 정의:**

- `개당금액` = 결제금액 ÷ 개수
- `최종개당금액` = 그룹 총 지출(배송비·할인 포함) × 해당상품 결제금액 비율 ÷ 해당상품 구매개수
  - 목적: 실제 지출 기준으로 얼마에 팔아야 순수익이 남는지 파악
- `추천판매가` = 두 가지 기준 동시 표시
  - 개당금액 기준: `개당금액 ÷ (1 - 마진율)`
  - 최종개당금액 기준: `최종개당금액 ÷ (1 - 마진율)`
  - 범위: 최소(마진율 15%) ~ 최대(마진율 50%), 설정에서 조정 가능

**상태 컬럼:**

- 재고 미반영: `[재고반영 버튼]` + 배경색 구분 표시
- 재고 반영완료: `[반영완료] + [반영취소 버튼]`
- 수정 모드 진입 시: 재고 관련 버튼 대신 `[저장] [취소]` 버튼으로 교체

**3dots 메뉴:**

- 수정 클릭: 해당 줄 전체가 input으로 전환 (인라인 편집)
- 삭제 클릭: 해당 목록 삭제. 재고반영 완료 상태면 삭제 불가 → 반영취소 후 삭제 가능

---

### 그룹 하단 요약

: 실시간 자동계산, 지출금액은 마이너스 빨간색 표시

```
총 주문금액   자동계산 (목록 결제금액 합산)     -000원
추가금        직접 입력 (예: 배송비)            -000원
할인금액      직접 입력 (예: 쿠폰)             +000원
────────────────────────────────────────────────────
총 지출       자동계산                         -000원
```

- 총 주문금액, 총 지출: 자동계산 (수정 불가)
- 추가금, 할인금액: 직접 입력 가능
- 총 지출 변경 시 → 그룹 내 전체 목록의 최종개당금액·추천판매가 실시간 재계산

---

### 그룹 하단 액션버튼

- `내역 추가`: **같은 결제날짜**로 라인 1건 더 등록 (등록 폼에 날짜 기본값 = 해당 그룹 날짜)
- `일괄 재고반영`: 그룹 내 미반영 목록 전체 재고반영 모달 순차 실행. 전체 반영완료 시 비활성화
- `일괄 주문취소`: 그룹 전체에 주문취소 처리
  - 재고반영된 목록이 있으면 → 해당 재고 전부 자동 차감 후 취소 처리
  - 그룹 전체에 마스크 레이어 + "주문취소" 텍스트 표시
  - 취소 상태에서 가능한 것: 그룹 삭제(완전삭제) / 주문취소 해제 / 드래그 순서이동
  - 주문취소 해제 클릭 시:
    - 마스크 제거
    - 이전에 재고반영됐던 목록 → 재고 자동 복구
    - 이전에 재고반영 안 됐던 목록 → 미반영 상태 그대로 복구
    - 토스트 알림: "주문취소가 해제됐습니다. 미반영 항목은 재고반영을 진행해주세요."

---

### 재고반영 모달

- 매입 목록 1줄당 1회 실행 (해당 목록의 수량 자동 입력)
- 상단: 카테고리 필터 select + 상품명 검색 input
- 상품 목록: SKU / 이미지 / 상품명 / 현재재고수
- 상품 미존재 시: 모달 내 `+ 상품 등록` 버튼으로 바로 등록 가능 → 등록 후 자동 선택
- 수량 조정: 자동입력된 수량 기준으로 `-` / `+` 버튼으로 조정
  - 추가(`+`) 클릭: 추가사유 입력
  - 차감(`-`) 클릭: 차감사유 입력
- SKU 연결 정보는 저장되지 않음 → 같은 상품 재매입 시 매번 검색·선택
- 확정 클릭 시: 해당 목록 상태 → 재고반영완료로 변경

    상품 변경 → 기존 상품 재고 복구 + 새 상품 재고 차감
    ```
- 삭제: 어떤 상태든 삭제 가능. 삭제 시 해당 주문 내 전체 상품 재고 자동 복구
- 주문취소: 배송 전 상태
- 반품: 반품 처리
- 교환: 상품 교환 처리

**취소/반품/교환 상태 처리:**

```
주문취소
  → 재고 자동 복구
  → 마스크 레이어 + "주문취소" 표시
  → 주문취소 해제 버튼 표시
  → 해제 클릭 시 재고 재차감
    (재고 부족 시 "재고가 없습니다" 알림 후 처리 중단)

반품
  → 마스크 레이어 + "반품중" 표시
  → 상단 반품완료 버튼 표시
  → 반품완료 클릭 시 전체 상품 재고 자동 복구 → "반품완료" 상태로 변경
  → 반품 해제 버튼 표시
  → 해제 클릭 시 재고 재차감
    (재고 부족 시 "재고가 없습니다" 알림 후 처리 중단)

교환
  → 상품목록 수정 모드 전환
  → 기존 상품 row: text-through + 배경색 빨간색 처리 (이력 보존)
  → 교환된 상품 row: 기존 상품 바로 아래에 표시
  → 저장 즉시 교환완료 확정
  → 기존 상품 재고 복구 + 교환 상품 재고 차감 동시 처리
    (교환 상품 재고 부족 시 "재고가 없습니다" 알림 후 처리 중단)
  → 교환은 1회만 가능
  → 교환완료 해제 버튼 표시
  → 해제 클릭 시 교환 상품 재고 복구 + 원래 상품 재고 재차감
    (재고 부족 시 "재고가 없습니다" 알림 후 처리 중단)
```

- 총 주문개수 / 총 주문금액

**상품 목록:**
- 채널SKU (없으면 `-`) / SKU / 상품명 / 수량

상품금액 합계     자동계산

- 할인            직접 입력
─────────────────────────
총 주문금액       자동계산
```

## 상품관리

---

### 왼쪽 — 상품 목록

- 상단 필터: 최근재고추가순 / 최신등록순 / 매출높은순 (전체 누적 판매금액 합산 기준)
- 검색: SKU, 상품명으로 검색
- 등록버튼
- 테이블: SKU / 상품명 / 재고개수 (페이지네이션)

**상품 등록 모달:**
- 입력항목: 카테고리(select) / 상품명 / 상품이미지
- SKU 자동생성: `카테고리코드-등록순번` (전체 상품 통합 순번)
  - 예: `TOY-0001`, `PKG-0002`
  - 등록 확정 전 미리보기 표시
  - 등록 후 수정 불가 → 등록 시 확인 모달 표시
    ```
    SKU를 확인해주세요
    TOY-0001
    등록 후 수정이 불가합니다.
    [취소] [확인]
    ```
- 삭제 조건: 매입·매출·재고 이력이 1건이라도 있으면 삭제 불가, 비활성화만 가능

**카테고리 관리 모달 (select 옆 + 버튼):**
- 기능: 추가 / 수정 / 삭제 / 비활성화
- 추가 시 입력항목: 카테고리명 / 코드 (영문 대문자 3자리 직접 입력)
  - 코드 중복 불가 → 중복 시 에러 표시
  - 코드 형식 검증: 영문 대문자 3자리만 허용
- 삭제 조건: 해당 카테고리로 등록된 상품이 있으면 삭제 불가 → 비활성화만 가능
- 비활성화: select 옵션에서 제외됨

---

### 오른쪽 — 상품 상세

**기본정보 (상단)**
- 표시항목: SKU / 카테고리 / 상품명 / 현재재고수
- 현재재고 표시 방식:
  - DB에 저장된 재고 수량 기준으로 표시
  - 기타 추가/차감 입력 시에만 UI상 실시간 미리보기 (저장 전 변경될 재고수 표시)
  - 저장 버튼 클릭 시 API 호출 → DB 저장 → 데이터 재조회 → 현재재고 업데이트
- 수정버튼: 카테고리 / 상품명만 수정 가능 (SKU 수정 불가)
- 삭제버튼: 이력 없을 때만 활성화

---

**채널 SKU 목록 (기본정보 아래)**
- 표시항목: 채널명 / SKU / 수정버튼 / 삭제버튼
  - 예: 네이버 / N-PROD-12345
- 추가버튼: 채널 select + SKU 직접 입력
  - 같은 채널에 SKU 중복 등록 불가
- 수정: 채널 / SKU 모두 수정 가능
- 삭제: 언제든 삭제 가능

**채널 관리 모달 (채널 select 옆 + 버튼):**
- 기능: 추가 / 수정 / 삭제 / 비활성화
- 카테고리 관리 모달과 동일한 구조
- 삭제 조건: 해당 채널로 등록된 SKU가 있으면 삭제 불가 → 비활성화만 가능
- 비활성화: select 옵션에서 제외됨

---

**재고 이력 (채널 SKU 목록 아래)**
- 상단 요약: 총 매입수 / 총 판매수
- 탭: 전체 / 매입 / 매출
- 추가버튼: 탭과 동일 선상, 탭 활성화와 무관하게 언제든 클릭 가능
  - 재고반영 모달과 동일한 구조
  - SKU select는 현재 상품으로 고정 및 수정 불가
  - 추가버튼으로 등록된 이력은 무조건 기타로 분류
  - 기타 이력: 수정 / 삭제 가능

**각 이력 아이템 표시:**
- 매입: `매입` / 날짜 / 구매처 / +개수 / 결제금액 / 개당금액
  - 클릭 시: 해당 매입 그룹 데이터 모달로 출력
- 매출: `매출` / 날짜 / 판매처 / -개수 / 결제금액
  - 클릭 시: 해당 매출 그룹 데이터 모달로 출력
- 기타: `기타` / 날짜 / 추가·차감 사유 / -/+개수
  - 수정 / 삭제 버튼
